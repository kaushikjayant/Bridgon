Capabilities

Everything You Need

Six integrated subsystems that work together or independently.

🔬
RTL Design Analysis
Full SystemVerilog/VHDL design elaboration via Verific. Extracts all module parameters with resolved values, types (integer, boolean, float, string, enumerated), and constraints from the complete design hierarchy. Replaces error-prone manual LEC log parsing with 100% accurate automated extraction.
🗺
Memory Map → Instance Map
Consumes a structured address map spreadsheet (XLSX) and auto-generates instance map (.imap) files with base addresses, slot sizes, and RTL parameter overrides for every IP instance. Handles complex scenarios: same IP instantiated multiple times with different parameter configurations.
🗄
Non-RTL Parameter Database
JSON-based ditabase for documentation-specific parameters (revision strings, security classifications, publishing modes) with chip_prefix-keyed hierarchical lookup. Version-control friendly, zero infrastructure dependency, directly consumable by all toolchain components.
🤖
AI-Assisted Chapter Generation
Claude API extracts SoC-specific content (clocking architecture, reset strategy, interrupt mapping, power management) from architecture specification PDFs and structures it as DITA concept/reference/task topics. Human review before inclusion ensures accuracy.
📋
Register Diagram Rendering
Transforms register database (.rdb) files into DITA reference topics with interactive SVG bit-field diagrams. Colour-coded access types: purple (R/W), green (RO), amber (W1C), red (WO). Structured tables with offset, field name, bit range, access type, and reset values.
🧠
RAG-Powered AI Chatbot
Builds a retrieval-augmented generation knowledge base from resolved DITA content. Users query documentation in natural language. The chatbot grounds responses in actual spec content with source citations. Reduces internal support queries by ~40%.
File Formats

Custom XML Vocabulary

Five purpose-built file formats that extend standard DITA/XML for semiconductor documentation.

.prm
Parameter Definition
Defines all configurable parameters (RTL & non-RTL) with defaults, constraints, type info, and source classification. Supports integer, boolean, float, string, and enumerated types.
.imap
Instance Map
Instantiates parameter values for a specific IP, SubSystem, or SoC publication instance. Links to .rdb files and carries per-instance RTL + non-RTL parameter overrides.
.rdb
Register Database
Defines register and bit-field descriptions in an IP-XACT-compatible XML format. Contains offset, size, access type, reset value, and per-bit-field descriptions.
.pub
Publishing Instructions
Defines register resolution groups, instance selectors, merging rules for multi-instance IPs, and output format configuration (PDF, WebHelp, RAG).
.ditamap
DITA Map
Standard DITA map extended with ipref/ssref links for IP and SubSystem references, and pm:cond_* attributes for parameter-driven conditional processing.

Example: Parameter Definition (.prm)

<!-- IP_FlexCAN.prm — Parameter definitions for FlexCAN IP --> <paramdef spec_id="IP_FlexCAN"> <parameter id="CAN_NUM_MB" class="integer" source="RTL"> <name>Number of Mailboxes</name> <default>64</default> <low_limit>16</low_limit> <high_limit>128</high_limit> </parameter> <parameter id="CAN_FD_ENABLE" class="boolean" source="RTL"> <default>true</default> <accepted_values>true,false</accepted_values> </parameter> <parameter id="CAN_SECURITY_CLASS" class="enumerated" source="nonrtl"> <default>CONFIDENTIAL</default> <accepted_values>PUBLIC,INTERNAL,CONFIDENTIAL,RESTRICTED</accepted_values> </parameter> </paramdef>
